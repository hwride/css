Floats

Floating left and right

Floating an element moves it to the side of a container. It also takes it out of flow, which is why the parent box doesn't expand to contain it.

a
float: left;
b
float: right;

Example with multiple floats. Note how they will appear out of order on the same line if floated in different directions.

a
float: left;
b
float: right;
c
float: left;
d
float: left;
e
float: right;

Floats and line boxes

Line boxes flow around floats. Block boxes ignore floats. See here the block container is behind the float but the line boxes flow around it:

a
width: 110px; float: left;

Same example with display: inline-block:

Clearing floats

You can stop an element being next to a float with clear: left/right/both. Note that you can only clear in a block formatting context, i.e. you can't clear inline elements.

a
float: left;
b
float: right;
clear: left;
Lorem ipsum dolor sit amet, consectetur adipiscing elit. vulputate.
a
float: left;
b
float: right;
clear: right;
Lorem ipsum dolor sit amet, consectetur adipiscing elit. vulputate.
a
float: left;
b
float: right;
clear: both;
Lorem ipsum dolor sit amet, consectetur adipiscing elit. vulputate.

Containing floats

A float is taken out of regular flow, so parents won't automatically expand to contain them:

a
float: left;

You can solve this in a few ways:

Clearfix
Using the clearfix hack. This involves adding an element after the float and setting it to clear the float:

a
float: left;

overflow: auto
Creating a new block formatting context using overflow: auto (a new BFC will contain everything inside it):

overflow: auto;
a
float: left;

display: flow-root
Creating a new block formatting context using the new and not as widely supported display: flow-root. This is a more semantic approach, and also won't result in undesired scrollbars from setting overflow when you don't really want overflow.

display: flow-root;
a
float: left;
