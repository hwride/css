Containing block

Reference: CSS 2.2 Visual formatting model details - Definition of "containing block"

Many properties in CSS are calculated according to a box's containing block. How to calculate this block is described here.

position: static, relative & sticky

For elements with a position of static, relative or sticky the containing block is formed by the edge of the content box of the nearest block container ancestor.

a
width: 800px; padding: 30px;
aa
width: 75%;
ab
position: relative; width: 75%

position: absolute

For elements with a position of absolute, the containing block is formed by the nearest ancestor that has a position of absolute, relative or fixed. For the following examples the block we're testing is green, semi transparent, has a dashed red border and set to top: 0; bottom: 0; left: 0; right: 0;.

position: absolute with block ancestor

The containing block is the edge of the padding box of the ancestor.

a
position: relative; width: 800px; padding: 30px;
b
width: 600px;

position: absolute with inline ancestor

The containing block is the bounding box around the padding boxes of the first and last inline boxes generated for the ancestor.

Parent is containing block

Here is an example where the the parent is the containing block:

position: relative; padding: 30px; padding: 20px; position: relative; padding: 30px; dddddddd

Grandparent is containing block

Here is an example where the grandparent is the containing block and has its own padding and multiple children:

position: relative; padding: 30px; padding: 20px; bbbbbbbbbb dddddddd

It seems to show the containing block is around the padding box around the ancestor, as opposed to the padding boxes of its children. Containing block split over multiple lines

If the inline element is split across multiple lines the containing block is undefined. On Chrome 80 the containing box appears to be the height of all line boxes, and the width up to the last line box. On Firefox 72 it appears to be the first line box. See:

padding: 20px; bbbbbbbbbbbbb cccccccccccc
