Inline and block

Block formatting context

  • In a block formatting context child block-level boxes are laid out vertically, one after the other.
  • A block container forms a block formatting context if it contains all block-level boxes.
  • If an element is a block container and a block-level items then it is known as a block box.

E.g. here a forms a block formatting context:

div a
div aa
div ab

Width, height, margin, padding and border

Width, height, margin, border and padding apply in all direction to block-level elements. Here ab has has them all applied:

div a
div aa
div ab
div ab

Anonymous block boxes

If a block container has a mix of inline and block elements as its children, the inline elements will be wrapped in anonymous block boxes, and a block formatting context will be formed. E.g. here a forms a block formatting context and ab is wrapped in an anonymous block box:

div a
div aa
span ab

Inline formatting context

A block container forms an inline formatting context if it contains all inline children. Inline children are laid out horizontally, one after the other. E.g. here a forms an inline formatting context:

div a span aa span ab

Line boxes

The box that surrounds a line of inline boxes is known as the line box. When several inline boxes cannot fit horizontally within a single line box, more vertically stacked line boxes are created to fit them.

div a span aa span ab span ac span ad

Line boxes are at least as tall as their largest inline box and can vary in height.

div a span aa span ab span ac span ad

When an inline box exceeds the width of a line box it is split into several boxes across several line boxes. If the inline box cannot be split, or is prevented from splitting by e.g. whitespace rules, then it overflows the line box.

div a span aa span ab span ac cannotbesplitsooverflowsssssssssssss more text span ad

Width, height, margins, border and padding

Width, height and vertical margins do not apply to inline-level elements. But horizontal margins, and border and padding in all directions do apply. Here ac has all properties applied, notice how it ignores width, height and vertical margins:

div a span aa span ab span ac span ad

Nested inline-level boxes

The contents of a regular inline-level box will participate in the same inline formatting context as its parent. E.g. here aba is a grandchild of a but it still participates in its inline formatting context:

div a span aa span ab span aba span ac

Anonymous inline boxes

Any text directly inside a block container is wrapped in an anonymous inline box.

div a Some text span aa some more text

Atomic inline-level box

If an inline-level box's contents does not participate in the inline formatting context of its parent then it is an atomic inline-level box. E.g. notice here how text does not flow between lines for the display: inline-block element:

div a span aa span ab some more text here span ac span ad
div a span aa span ab display: inline-block span ac span ad
