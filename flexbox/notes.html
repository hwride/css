Flexbox

Browser support: IE10 & above

What flexbox can control

  • Direction - The direction elements are laid out.
  • Alignment - horizontal and vertical alignment of elements.
  • Flexibility - what proportion of size elements take up, whether they can grow or shrink.
  • Ordering - the order of elements, this can be changed.

Flexbox axis

Flexbox has two axis which it applies some of its properties to:
flex-direction: row flex-direction: column Properties that apply to this axis
Main axis Horizontal Vertical justify-*
Cross axis Vertical Horizontal align-*

CSS rules

Flexbox containers

  • display
    • display: flex - Element behaves like a block element and lays content out according to the flexbox model.
    • display: inline-flex - Element behaves like an inline element and lays content out according to the flexbox model.
  • flex-direction: row, column, row-reverse, column-reverse
  • flex-wrap: wrap
  • flex-flow: <flex-direction> <flex-wrap> - Shorthand property
  • justify
    • justify-content
    • justify-items
  • align
    • align-content
    • align-items
  • place-[content|items]: <align-value> <justify-value> - A shorthand for setting both justify and align.

Flexbox items

  • flex-grow: 1 - If extra space is left in the container, the proportion this flex item should take.
  • flex-shrink: 1 - If items are overflowing, the proportion this flex item should shrink
  • flex-basis: 200px - The initial size for the flex item.
  • flex: <flex-grow> <flex-shrink> <flex-basis> - Shorthand property. Can leave out arguments.
  • justify-self - As above but for an item.
  • align-self - As above but for an item.
  • place-self - As above but for an item.

Questions

  • How does flexbox affect its child items if nothing is changed except display: flex?
  • How does flex-shrink work with flex-wrap? If items can wrap and still be inside the container then it will have no effect. But if an item is larger than actual flex container then having flex-shrink set will allow the item to shrink down to the container width. But wrap apparently takes priority over shrink.
